Detección automática

El motor de CSV detecta encoding y delimitador sin configuración manual:

  • Encoding: UTF-8, UTF-8 con BOM, UTF-16 LE, UTF-16 BE.
  • Delimitador: ;, ,, TAB (selecciona el más frecuente en la cabecera, respetando comillas).
  • Cabeceras: lookup case-insensitive contra listas de alias por columna.

Si la cabecera no coincide con ningún alias y el CSV tiene al menos 2 columnas, se asume que la primera es el campo principal y la segunda el secundario, con un aviso en el log.

Esquema EmailId

Usado por Agregar miembros a grupo/DL. Cada fila representa un usuario a añadir.

Columnas

Campo lógicoAlias aceptadosObligatorio
Email Email, Mail, UserPrincipalName, UPN, Correo, PrimarySmtpAddress, WindowsLiveID, Login, EmailAddress Al menos uno de Email o Id
Id Id, ID, ObjectId, ExternalDirectoryObjectId, Guid, GUID, EntraId, AzureObjectId Al menos uno de Email o Id

Ejemplo mínimo (solo Email)

Email
juan.perez@contoso.com
maria.gomez@contoso.com

Ejemplo completo

Email;Id
juan.perez@contoso.com;12345678-90ab-cdef-1234-567890abcdef
maria.gomez@contoso.com;
;abcdef12-3456-7890-abcd-ef1234567890
Resolución inteligente: si falta una columna, se intenta resolver vía Microsoft Graph usando la otra. Si ambas faltan o no resuelven, la fila se marca como ID_NO_RESUELTO en el resultado.

Esquema EmailGroupName

Usado por Crear grupos/DL desde CSV. Cada fila representa un miembro a añadir a un grupo (que se crea si no existe).

Columnas

Campo lógicoAlias aceptadosObligatorio
Email Email, Mail, UserPrincipalName, UPN, Correo, PrimarySmtpAddress, WindowsLiveID, Login, EmailAddress, MemberEmail Sí
GroupName GroupName, Group, Grupo, NombreGrupo, ListName, DL, DistributionList, GroupAlias, Nombre Sí (heredable de fila previa si está vacía)

Ejemplo

GroupName;Email
ventas-norte;juan.perez@contoso.com
ventas-norte;maria.gomez@contoso.com
ventas-sur;pedro.lopez@contoso.com

El dominio del grupo se solicita al ejecutar el script (no se incluye en el CSV). En el ejemplo anterior, con dominio contoso.com, se crearán los grupos ventas-norte@contoso.com y ventas-sur@contoso.com.

Herencia de GroupName: si una fila tiene GroupName vacío, hereda el valor de la fila anterior. Útil para CSVs agrupados visualmente.

Múltiples emails por fila

En el esquema EmailId, si una celda Email contiene varios correos separados por espacios, saltos de línea o comas, se expanden automáticamente a una fila por correo (manteniendo el mismo Id si existe).

Email;Id
juan.perez@contoso.com maria.gomez@contoso.com;

Se procesará como dos filas independientes.

Encoding y delimitadores

Encoding recomendado

UTF-8 con BOM garantiza interoperabilidad con Excel y otros editores. UTF-8 sin BOM también funciona pero algunos editores pueden mal-interpretar acentos.

Delimitador recomendado

Punto y coma (;) es estándar en locales europeos y evita conflictos con comas dentro de campos.

Campos con caracteres especiales

Encerrar entre comillas dobles si contienen el delimitador, comillas dobles o saltos de línea:

Email;Id
"valor,con,comas@contoso.com";
"valor con ""comillas""@contoso.com";
Atención: evitar caracteres de control (\0, \x01-\x1F) excepto TAB. La detección de delimitador asume contenido textual ASCII/UTF-8.

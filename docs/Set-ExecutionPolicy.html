Configurar ExecutionPolicy

PowerShell exige una política de ejecución que permita scripts locales. GREX365 intenta establecerla automáticamente, pero algunos entornos corporativos lo bloquean.

Estado actual

Comprueba la política activa por scope:

Get-ExecutionPolicy -List

Salida típica en una instalación recién hecha:

        Scope ExecutionPolicy
        ----- ---------------
MachinePolicy       Undefined
   UserPolicy       Undefined
      Process       Undefined
  CurrentUser       Undefined
 LocalMachine    Restricted

Si LocalMachine es Restricted, los scripts no se pueden ejecutar a menos que se levante la política para el usuario actual o el proceso.

Opciones recomendadas

Opción 1 — Process scope (recomendado)

Levanta la restricción solo para la sesión actual. No afecta otras sesiones ni a otros usuarios:

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass -Force
GREX365 ejecuta esta línea automáticamente al arrancar. Solo necesitas hacerlo a mano si el bootstrap falla por GPO.

Opción 2 — CurrentUser scope

Persistente entre sesiones para tu usuario:

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

RemoteSigned permite scripts locales sin firma y exige firma para scripts descargados (más estricto que Bypass).

Opción 3 — Ejecutar el launcher con override inline

powershell.exe -ExecutionPolicy Bypass -File ".\Main.ps1"

Útil para shortcuts o scheduled tasks sin tocar configuración global.

Niveles de política

ValorComportamiento
RestrictedNingún script se ejecuta. Default histórico en algunas ediciones.
AllSignedSolo scripts firmados por un publisher de confianza.
RemoteSignedScripts locales libres; scripts descargados (zone Internet) requieren firma.
UnrestrictedCualquier script, pero advierte para los descargados.
BypassSin restricciones ni advertencias. Recomendado para Scope Process.

Errores comunes

cannot be loaded because running scripts is disabled on this system

Tu política activa es Restricted. Aplica Opción 1 o 2 arriba.

cannot be loaded. The file ... is not digitally signed

Política AllSigned activa. O cambia a RemoteSigned en CurrentUser, o desbloquea el archivo si fue descargado:

Unblock-File -Path .\Main.ps1

GREX365 hace Unblock-File recursivo en sus propios scripts durante el bootstrap.

GPO o MDM bloquean el cambio

En tenants con política administrada, Set-ExecutionPolicy devolverá un error. Usa la Opción 3 (override inline en el launcher) — no requiere permisos para modificar configuración persistente.

Atención: nunca uses Set-ExecutionPolicy -Scope LocalMachine sin justificación corporativa. Afecta a todos los usuarios del equipo y puede violar políticas de cumplimiento.

01General

Los scripts de GREX365 leen ficheros .csv con dos esquemas de columnas según el módulo:

SchemaMóduloColumnas
A Inyección de usuarios a grupo / DL existente Email obligatoria · Id opcional
B Creación masiva de grupos / DL desde CSV Email obligatoria · GroupName obligatoria
Detección automática. Encoding (UTF-8 con/sin BOM, UTF-16) y delimitador (, ; TAB) se autodetectan. No hace falta declararlos.

02Schema A — Email [+ Id]

Para módulos que añaden miembros a un grupo o lista de distribución existente.

Cabeceras válidas

CampoAlias aceptadosEstado
EmailEmail · Mail · UserPrincipalName · UPN · Correo · PrimarySmtpAddress · EmailAddress · MemberEmailobligatoria
IdId · ObjectId · UserId · ExternalDirectoryObjectIdopcional

Ejemplo mínimo (sólo Email)

Email
ana.garcia@contoso.com
luis.perez@contoso.com
maria.lopez@contoso.com

Ejemplo completo (Email + Id)

Email;Id
ana.garcia@contoso.com;9c0f8b21-43a1-4e7c-9d2f-1a2b3c4d5e6f
luis.perez@contoso.com;a1b2c3d4-5e6f-7890-abcd-ef1234567890
Comportamiento. Si falta el Id, el script lo resuelve consultando Microsoft Graph por el correo. Si el correo y el Id están ambos presentes, el Id manda en caso de conflicto.

Una sola columna sin cabecera

ana.garcia@contoso.com
luis.perez@contoso.com
maria.lopez@contoso.com

Si no hay cabecera, la primera columna se interpreta como Email.

03Schema B — Email + GroupName

Para crear varios grupos en lote y poblarlos en una sola pasada. Cada fila asocia un usuario a un grupo. El script agrupa internamente por GroupName.

Cabeceras válidas

CampoAlias aceptadosEstado
EmailMismos alias que Schema Aobligatoria
GroupNameGroupName · Group · Grupo · NombreGrupo · ListName · DL · DistributionList · GroupAlias · Nombreobligatoria

Ejemplo

Email;GroupName
ana.garcia@contoso.com;Marketing-Lisboa
luis.perez@contoso.com;Marketing-Lisboa
maria.lopez@contoso.com;Ventas-Madrid
pedro.ruiz@contoso.com;Ventas-Madrid
Construcción del email del grupo. El script pide un dominio al inicio (ej: contoso.com) y construye el correo del grupo como GroupName@dominio. El GroupName se usa también como DisplayName y MailNickname.

Filas con GroupName vacío

Email;GroupName
ana.garcia@contoso.com;Marketing-Lisboa
luis.perez@contoso.com;
maria.lopez@contoso.com;Ventas-Madrid

Si una fila trae GroupName vacío, hereda el último valor no vacío de las filas anteriores (en el ejemplo, Luis cae en Marketing-Lisboa).

04Delimitadores y encoding

Delimitadores soportados

  • Coma ,
  • Punto y coma ; — recomendado en entornos europeos por compatibilidad con Excel ES/PT
  • Tabulador TAB

Encoding

  • UTF-8 (con o sin BOM) — recomendado
  • UTF-16 LE / BE (BOM detectado)

Comillas

Los valores entre comillas dobles se respetan. Para escapar una comilla dentro de un valor, dúplicala:

Email;DisplayName
ana@contoso.com;"García, Ana"
luis@contoso.com;"O""Brien, Luis"

05Validaciones

ReglaComportamiento
Email vacíoFila omitida, registrada en log como Error / Email vacío
Email malformadoFila omitida, registrada como Error / Formato de email inválido
Usuario no existe en GraphFila omitida, registrada como Error / Usuario no encontrado
Miembro ya perteneceFila omitida, registrada como MemberSkipped
Grupo ya existe (Schema B)Se reutiliza, no se duplica. Se añaden miembros nuevos.
GUID inválido en columna IdSe ignora el Id, se cae a resolución por correo
Validador de email. Patrón aplicado: ^[^@\s]+@[^@\s]+\.[^@\s]+$. No valida MX ni existencia real, sólo formato.

06Errores comunes

1. Excel guarda con BOM y separador local

No es problema: ambos se autodetectan. Sólo cuida que la cabecera empiece con un alias válido (no la primera celda en blanco).

2. Cabecera con espacios o tildes

Las cabeceras se comparan en minúsculas y sin acentos. Correo, correo y CORREO son equivalentes.

3. Fila vacía intermedia

Las filas en blanco se ignoran sin error.

4. CSV con sólo una columna sin cabecera

Schema A: la columna se asume Email. Schema B: necesita siempre Email + GroupName; con sólo una columna falla.

5. Caracteres invisibles (BOM duplicado)

El script limpia el BOM al inicio de la primera celda. Si tu cabecera aparece como ﻿Email en logs, el script ya lo está saneando.

Ejemplos malos

// Sin cabecera y dos columnas (Schema B falla):
ana@contoso.com;Marketing-Lisboa
luis@contoso.com;Marketing-Lisboa

// Cabecera incorrecta en Schema B:
correoUsuario;listaDistribucion
ana@contoso.com;Marketing-Lisboa

// Email malformado:
ana_at_contoso.com;Marketing-Lisboa
